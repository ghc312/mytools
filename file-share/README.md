# 📁 FileShare — 本地文件分享

在任意目录启动 HTTP 服务器，自动检测局域网 IP，终端显示二维码。手机扫码即可浏览、下载文件，免去微信传文件的压缩和大小限制。

## 快速开始

```bash
# 安装依赖（可选，仅二维码功能需要）
pip install qrcode

# 分享当前目录
python share.py

# 分享指定目录
python share.py ~/Downloads

# 指定端口 + 允许上传
python share.py ~/Photos -p 3000 -u
```

扫码后手机上看到的界面：

- 📂 文件夹和文件列表（目录优先排列）
- ⬇ 一键下载按钮
- 📋 复制链接按钮
- 📤 拖拽上传（需 `-u` 开启）

## 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `directory` | 要分享的目录 | 当前目录 |
| `-p, --port` | HTTP 端口 | 8080 |
| `-u, --upload` | 允许手机上传文件 | 关闭 |
| `--no-qr` | 不显示终端二维码 | 显示 |
| `--qr-png PATH` | 保存二维码为 PNG 图片 | 不保存 |
| `--no-browser` | 不自动打开浏览器 | 自动打开 |

## 使用场景

| 场景 | 命令 |
|------|------|
| 电脑文件发到手机 | `python share.py ~/Documents` → 扫码下载 |
| 手机照片传到电脑 | `python share.py ~/Photos -u` → 手机上传 |
| 局域网内多设备传文件 | 起服务，所有人都能访问 |
| 大文件分享 | 无大小限制，不压缩 |

## 安全说明

- **默认只读**，上传需手动开启 `-u`
- 只分享指定目录，无法访问父目录
- 仅局域网可访问（防火墙内）
- 按 `Ctrl+C` 立即停止服务

## 技术细节

- Python 标准库实现，零外部依赖（二维码用 `qrcode` 可选）
- 支持 Range 请求，大文件和视频可断点续传/拖拽播放
- 中文文件名自动 RFC 5987 编码
- 响应式 Web UI，手机/电脑都友好
