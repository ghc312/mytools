# 豆包消息批量清理工具

> 一键清空豆包网页版（doubao.com）主对话中的所有消息，无需手动逐条删除。

## 功能

- **全自动批量删除**：自动滚动加载历史消息，分批调用豆包官方删除接口
- **无需账号密码**：全程在你自己电脑的浏览器里运行，不经过任何第三方
- **自动参数探测**：自动从浏览器环境中读取 `device_id` / `web_id` 等必要参数
- **智能重试**：历史消息加载慢时自动指数退避等待，最长等 3 分钟
- **自动确认**：自动点击豆包弹出的"删除确认"按钮

## 使用方法

### 准备工作

- 一台电脑（Mac / Windows 均可）
- Chrome 或 Edge 浏览器
- 已登录豆包网页版

### 操作步骤

1. 打开要清理的对话页面
   ```
   https://www.doubao.com/chat/XXXXXXXXXXX
   ```

2. 按 `F12` 打开开发者工具，切换到 **Console（控制台）** 标签

3. 将 `delete-doubao-console.js` 的**全部内容**复制粘贴到控制台，按回车运行

4. 等待脚本自动完成：
   ```
   🚀 开始批量删除...
   第 1 轮: 发现 52 条未删除消息...
   ✅ 已删除 20 条（累计 20 条）
   ✅ 已删除 20 条（累计 40 条）
   ...
   🎉 全部完成！共删除 2386 条消息
   ```

### 运行示例

```javascript
// 完整脚本见 delete-doubao-console.js
// 粘贴到豆包对话页面的 Console 中运行即可
```

## 工作原理

```
用户打开豆包对话页面
        │
        ▼
粘贴脚本到浏览器 Console
        │
        ▼
┌──────────────────────────┐
│ ① 自动探测设备参数        │  ← 从 performance API / storage / window 读取
│   device_id, web_id 等    │
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ ② 滚动到对话顶部          │  ← 触发豆包加载历史消息
│   等待消息渲染到 DOM       │
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ ③ 从 DOM 提取消息 ID      │  ← 读取 [data-message-id] 属性
│   去重、过滤已删除的       │
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ ④ 调用删除接口            │  ← POST /im/message/batch_update_status
│   每批 20 条，间隔 500ms   │     cmd=2210, status=5
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ ⑤ 重复 ②~④，直到无新消息  │  ← 连续 15 轮无新消息则结束
│   指数退避最多等 3 分钟    │
└──────────┬───────────────┘
           ▼
       🎉 完成！
```

### 删除接口说明

脚本调用的是豆包自己的删除接口，和你手动点击"删除"按钮调用的接口完全一致：

| 参数 | 值 | 说明 |
|------|-----|------|
| 接口路径 | `/im/message/batch_update_status` | 批量更新消息状态 |
| cmd | `2210` | 命令码 |
| status | `5` | 状态值，5 = 删除 |
| conversation_type | `3` | 对话类型 |

## 配置说明

脚本顶部有一个 `CONFIG` 对象，可根据需要调整：

```javascript
const CONFIG = {
  batchSize:      20,       // 每批删除条数（减少可降低风控风险，建议 10~30）
  batchDelay:     500,      // 批次间隔毫秒（增大可降低风控风险，建议 ≥300）
  scrollDelay:    1500,     // 滚动后等待加载的初始毫秒数
  maxNoNewRounds: 15,       // 连续多少轮无新消息才终止
  retryBaseDelay: 2000,     // 指数退避基础延迟毫秒
  retryMaxDelay:  180000,   // 指数退避最大延迟毫秒（3 分钟）
};
```

## 常见问题

### Q: 运行后提示"未能自动探测到 device_id / web_id"

A: 极少情况。确保你在豆包对话页面（URL 包含 `/chat/`）运行，且页面已完全加载。如果仍然失败，刷新页面后重试。

### Q: 删到一半停了怎么办？

A: 脚本支持断点续删——重新粘贴运行即可，已删除的消息不会重复处理。

### Q: 会封号吗？

A: 不会。脚本调用的是豆包官方的删除接口，和你手动逐条删除没有区别，只是替你做了重复劳动。

### Q: 对话本身会被删除吗？

A: 不会。只删除对话里的消息，对话和豆包机器人本身不受影响。

### Q: 删除可以恢复吗？

A: **不可恢复**。运行前请确认你确实要清空这些对话内容，建议先截图保存重要信息。

### Q: 手机端能用吗？

A: 目前只支持电脑端浏览器。手机浏览器无法打开开发者控制台。
